What is good code?

Good code is code which is easy to understand and easy to change

Assuming the code already does what we want, has the necessary performance characteristics and is not buggy.

Why write good code?

  • Write once, read and modify many times.
  • Less money and time spent on development.
  • Best code is the one not written.

What's wrong with the code?

  • No doc
  • Bad layout
  • Input variables are changed (inputRec)
  • Does too much

So how do we measure good code?

Code fearlessly

  • All the code needs to be versioned
  • Do developemnt on development machines (not on production)
  • TDD creates a safety net

Joel Test

  • Do you use source control?
  • Can you make a build in one step?
  • Do you make daily builds?
  • Do you have a bug database?
  • Do you fix bugs before writing new code?
  • Do you have an up-to-date schedule?
  • Do you have a spec?
  • Do programmers have quiet working conditions?
  • Do you use the best tools money can buy?
  • Do you have testers?
  • Do new candidates write code during their interview?
  • Do you do hallway usability testing?

What does a good project setup look like?

  • Version control
  • Documentation exists
  • Specifications are kept up to date
  • README.md exists
  • Instructions on how to setup the project exist
  • Development happens locally
  • Backup process works
  • Syntax checking and tests (in the cloud)
  • Automatically deployed/built to (dev/alpha) on push
  • Code review with approval before deploy
  • Run security software
  • Same process to deploy to production
  • Process is triggered by the press of a button
  • Changelog exist
  • SymVer - google it
  • Everybody has permissions to do everything

It is your job to make sure a project has all this shiznit
